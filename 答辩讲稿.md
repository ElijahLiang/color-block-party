# 《异质同构 · 与蒙德里安》答辩讲稿

> 一句话定位:这不是预先做好的动画,而是一个**"边听边画"的实时系统**——
> 音乐一播放,程序就实时"听懂"它,再把听到的东西翻译成会动、会闪、会变色的三维画面。
> 同一首歌每次播放都略有呼吸般的不同,因为分析是实时发生的。

---

## 一、30 秒电梯陈述(开场就说这段)

"我们做的是一个音乐可视化海报。它的核心是:**先用信号处理算法把一段音乐实时拆解成节奏、能量、音高、和弦这几个维度,再把每个维度映射成画面里的一种视觉语言**——比如低音的力度变成方块的爆裂,和弦的调性变成画面的主色。我们借用蒙德里安的红黄蓝几何作为画布,让'声音→颜色→运动'形成一种**异质同构**:不同媒介、同一种结构。"

---

## 二、用了哪些算法?(评委问"技术含量"时回答)

所有分析都跑在浏览器的 Web Audio 上,实时进行。用一句话解释每个算法:

| 算法 | 通俗解释 | 在作品里干什么 |
|---|---|---|
| **FFT 快速傅里叶变换** | 把声波从"波形"变成"频谱"——即每一瞬间**各个音高分别有多响** | 一切分析的基础 |
| **RMS 响度 + 自适应基线** | 算整体音量,并和"最近的平均"比,判断"比平时响了多少" | 画面的总强弱 |
| **能量突变检测(重拍)** | 低频能量突然蹿高一截就算一次重拍 | 打节奏点 |
| **谱通量 Spectral Flux + 自适应阈值(底鼓)** | 专抓"咚"那一下的**攻击瞬间**,持续的低音不会误判 | 触发爆裂 |
| **8 段感知频段划分** | 把频谱按人耳习惯切成 8 条(低→高),低频分得细一些 | 频谱"立"成画面 |
| **YIN 音高检测** | 在时域上估计主旋律此刻唱/弹的是哪个音 | 识别旋律基频 |
| **Chroma + 和弦模板匹配** | 把能量折叠到 12 个音级,匹配大/小三和弦,判断**当前和弦与调性** | 决定画面主色 |
| **副歌检测** | 人声能量突增 + 音量占比上升 → 判断进入高潮段 | 全场最"嗨"的状态 |

> 重点话术:"我们没有用现成的可视化插件,而是**自己实现了这一整套从频谱到节拍、到和弦的分析链**。"

## 三、一段音乐被"折"成了哪几部分?

我们把一段连续的音乐,实时拆成 **5 个层次**(可以画在 PPT 上):

1. **整体响度层** —— 音乐的"呼吸":此刻是强是弱。
2. **节奏层** —— 重拍 + 底鼓打击点(音乐的"骨架/脉搏")。
3. **频率层** —— 把声音从低到高切成 **8 个频段**:最低是底鼓贝斯,中间是人声,最高是镲片高频。
4. **和声层** —— 主旋律音高 + 当前和弦(决定情绪与调性色彩)。
5. **段落层** —— 副歌/高潮的识别(整首歌的"起伏结构")。

一句话:**横向看是节奏,纵向看是频率,深层看是和声,宏观看是段落,全局是响度。**

## 四、这几部分怎么变成画面?(逐一对应,讲解主力)

画面主体是一团漂浮的**蒙德里安方块云**,外面套着线框外壳,打击时迸发碎片和烟花。映射关系如下:

### ① 整体响度 → 画面的"总开关"
音乐安静或停下时,画面收敛成一张**安静的几何海报**;音乐一起来,整个场景才"活"过来——所有的运动幅度都被它控制。
> 这点很重要:**它不是一直在乱动,而是跟着音乐的能量"该静则静、该动则动"。**

### ② 8 个频段 → 方块阵按高度跳舞(全场最核心的映射)
方块按**高度从下到上,对应低音到高音**。哪个频段一响,**那一层的方块就放大、发亮、跳动**。
> 等于把**频谱竖起来在跳舞**——你听到贝斯,底部一排就鼓动;高频镲片一响,顶部就闪烁。

### ③ 底鼓"咚" → 爆裂 + 烟花 + 镜头吸气 + 闪光
每一记重底鼓:整团方块**向外炸开、再被弹簧吸回**(像心跳/爆裂),同时**迸发碎片和烟花**,镜头猛地"吸一口气"(视野收一下),整个画面**泛光闪亮一下**。
> 这是观众最能直接"看到鼓点"的地方。

### ④ 和弦 / 调性 → 画面主色(作品立意的点睛之笔)
当前是什么和弦,就决定这一刻迸发的碎片和烟花用**红、黄、还是蓝**——正好是蒙德里安的三原色。
> 这就是标题"**异质同构**":听觉里的和声结构,被同构地映射成视觉里的颜色结构。

### ⑤ 重拍 → 方块颜色轻轻闪动(踩在节奏点上的微妙律动)。

### ⑥ 副歌 / 高潮 → 全场最"嗨"
进入副歌时:**持续不断地撒出碎片**,画面更亮、泛光更强,**相机加速漫游、视野收紧**,营造高潮的张力。

### ⑦ 旋律音高 → 我们也实时检测了主旋律基频与和弦根音,用来驱动上面的颜色选择(和声 → 色彩)。

## 五、收尾立意(打动评委的一段)

"蒙德里安用红黄蓝和直线,把世界抽象成最纯粹的秩序。我们做的,是让**声音也走进这套秩序**:节奏成为脉搏,频率成为高低,和声成为色彩,响度成为呼吸。
因为分析是**实时**的,所以它是一个活的系统——**音乐在演奏,画面也在演奏**,二者是同一段情绪的两种媒介。这,就是'异质同构'。"

## 六、评委可能追问 & 回答预案

- **Q:这是预渲染的视频吗?**
  A:不是。是浏览器里实时分析、实时渲染的,换任何一首歌都能即时驱动;现在就可以现场插一首试。

- **Q:怎么保证鼓点不乱触发?**
  A:底鼓用的是谱通量配**自适应阈值**(均值+标准差)并加冷却时间,所以它对不同响度的歌会自校准,持续低音也不会误判成打击。

- **Q:颜色为什么这么选?**
  A:我们把 12 个音级映射到蒙德里安的红黄蓝三原色,由当前**和弦根音**决定主色——是有音乐逻辑的,不是随机变色。

- **Q:用了什么框架?**
  A:画面是 Three.js(WebGL)做的三维粒子/实例化渲染加泛光后期;音频分析是基于 Web Audio API **自己实现**的整套算法。
