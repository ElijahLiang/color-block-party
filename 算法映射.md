# 算法映射 · 音乐 → 视觉

这个海报的核心是一条「**音频分析 → 特征驱动 → 三维视觉**」的实时管线,分两层:

- **`music.js`** — 分析层。用 Web Audio 的 `AnalyserNode` 把声音拆成一组**特征量**(节拍、底鼓、8 频段能量、副歌、音高、和弦……),挂在 `window.MUSIC.state` 上。
- **`poster-vj.js`** — 映射层。每帧读这些特征,先做**包络平滑**,再驱动 Three.js 场景里的方块云、碎片、烟花、外壳、相机和泛光。

每帧顺序(`poster-vj.js` 的 `animate()`):
`MUSIC.update()`(重算特征) → `readMusic()`(特征→包络) → `updateCubes/Shards/Fireworks` → 相机/外壳/泛光 → `composer.render()`。

---

## 一、分析层:音频 → 特征(`music.js`)

FFT 大小 `4096` → 2048 个频点,约 **10.7 Hz/点**;`smoothingTimeConstant=0.4`(故意偏低,保住瞬态给底鼓检测用)。

| 特征 | 字段 | 怎么算 |
|---|---|---|
| 整体响度 | `level` / `levelSmooth` / `levelAvg` | 时域 RMS;快平滑(0.35)给瞬时感,慢平滑(0.06)做自适应基线 |
| 低/中/高频 | `bassEnergy` / `vocalEnergy` / `backingEnergy` | 频谱按 `<500Hz` / `500–4000Hz` / `>4000Hz` 三段求和归一化 |
| **8 个感知频段** | `bandVal[8]` / `bandExcess[8]` | 频点按 `(i/8)²` 幂曲线切分(低频分到更少点,符合听感),逐段平滑 |
| **节拍 Beat** | `beat` / `beatPulse` | 低频能量 `> 1.5×` 自适应均值,且过冷却期 → 触发 |
| **底鼓 Kick** | `kick` / `kickPulse` / `kickEnergy` | 对 `<160Hz` 的**浮点频谱做谱通量(spectral flux)**,自适应阈值 `均值 + 2×标准差`,带绝对地板和 120ms 冷却 → 只在「攻击/打击」瞬间触发,持续低音不会误触 |
| **副歌 Chorus** | `chorusLevel`(0..1) | `chorusScore = 0.5×人声突增 + 0.3×音量比 + 0.2×人声占比`,过门后用保持帧 + 缓升缓降做成平滑包络 |
| 音高(单音) | `pitchHz` / `pitchMidi` / `pitchClarity` | 时域 **YIN** 算法估主旋律基频 |
| **和弦** | `chord.root` / `chord.quality` | 12 维 chroma 上做大/小三和弦模板匹配(按「显著度」打分),带迟滞防抖 |
| 驱动量 | `gain` / `timeScale` | 副歌时 `gain` 缓慢累积;`timeScale = 1 + chorusLevel×0.35`(副歌让时间略微加速) |

## 二、映射层:特征 → 包络(`poster-vj.js` 的 `readMusic`)

视觉不直接吃原始特征,而是先转成**平滑包络**,避免抖动。关键是一个**总门控 `motionGate`**:

```
levelEnv    = lerp→ clamp01(levelSmooth × 3.2)      // 整体动感
bassEnv     = lerp→ clamp01(bassEnergy × 4.5)       // 低频冲击
chorusEnv   = lerp→ clamp01(chorusLevel)            // 副歌强度(慢)
bandEnv[b]  = lerp→ clamp01(bandVal[b] × 3.0)       // 8 段各自的能量
motionGate  = lerp→ clamp01((levelEnv − 0.22) / 0.45)   // ★ 安静时≈0,响时≈1
kickEnv     // 命中强底鼓置 1,每帧 −0.05 衰减
beatEnv     // 命中节拍置 1,每帧 −0.07 衰减
```

> **`motionGate` 是灵魂**:几乎所有「运动幅度」都乘上它,所以音乐弱/停时画面自动收敛成一张安静的海报,响起来才整体「活」过来。

**强底鼓判定**(同时满足才算一次视觉爆发,避免太频繁):
`m.kick` 且 `levelEnv>0.28` 且 `(bassEnv>0.18 或 kickEnergy>0.08)` 且 距上次 `>0.32s`。

## 三、特征 → 视觉元素(逐个)

### 1) 蒙德里安方块云(主体,9×13×5=585 个 `InstancedMesh`)
弹簧物理把每个方块拉回原位;**方块按 Y 坐标分到 8 个频段**(`bandOf = floor(y/GY × 8)`,**底部=低频、顶部=高频**),所以频谱直接「画」在了方块阵的高度上:

```
本格能量 band = bandEnv[bandOf[k]]
缩放 scale   = CUBE × (0.62 + band×1.15 + kickEnv×0.45 + levelEnv×0.25)
漂浮幅度 live = (0.16 + band×0.6 + chorusEnv×0.35) × motionGate
自转速度     = (0.25 + band×0.7 + chorusEnv×0.5) × motionGate
高频微抖 micro= (0.01 + levelEnv×0.03 + kickEnv×0.02) × motionGate
颜色亮度 flash= 基色 × (0.76 + band×0.44 + kickEnv×0.3 + beatEnv×0.1 + chorusEnv×0.16)
```

### 2) `detonate()` — 底鼓「引爆」
强底鼓时给所有方块一个**沿径向向外的冲量**:`力 ∝ 0.55 + bassEnv×0.85 + chorusEnv×0.6`,随后弹簧把它们吸回 → 整团方块「炸开又聚拢」的呼吸感。

### 3) 碎片 Shards(混合几何体爆发)
- **强底鼓**:`spawnShards(数量 = 26 + (bassEnv+chorusEnv)×38, 速度 = 6 + bassEnv×5 + chorusEnv×4)`
- **副歌持续**:`motionGate>0.35 且 chorusEnv>0.45` 时按 `chorusEnv` 持续撒出碎片
- 碎片有重力、阻力、寿命,抖动幅度随 `levelEnv + kickEnv` 增强

### 4) 烟花 Fireworks(加法混合粒子)
强底鼓且 `bassEnv>0.2 或 chorusEnv>0.3` 时迸发:
`数量 = 150 + bassEnv×110 + chorusEnv×110`,粒子受重力成拱形轨迹并闪烁。

### 5) 颜色 ← 和弦/音高(★ 和声映射)
`PITCH_COLOR[12]` 把 12 个音级映射到 **3 种辉光色(红/黄/蓝)**。强底鼓时取**当前和弦根音**决定本次碎片+烟花的颜色(无和弦则随机):
```
idx = chord.root≥0 ? PITCH_COLOR[chord.root] : random(0..2)
```
→ **和声内容决定画面主色**,这是「异质同构」的关键:听觉的调性 ↔ 视觉的色彩。

### 6) 外壳 Shell(线框二十面体)
```
旋转速度 += dt × (0.025 + levelEnv×0.18 + chorusEnv×0.16) × motionGate
缩放      = 1 + kickEnv×0.1 + chorusEnv×0.06
透明度    = 0.1 + levelEnv×0.16 + chorusEnv×0.12
```

### 7) 相机(漫游 + 呼吸)
```
环绕角速度 += dt × (0.04 + levelEnv×0.08 + chorusEnv×0.08) × motionGate
半径        随副歌收紧;位置/滚转重度平滑(指数缓动)避免抖
FOV         = 74 − kickEnv×3 − chorusEnv×3   // 底鼓/副歌时镜头「吸气」
```
音乐停止时相机回到一个**略带角度的静态构图**,让海报读起来干净。

### 8) 泛光 Bloom / 色调(后期)
```
bloom.strength       = 0.4 + kickEnv×0.2 + chorusEnv×0.24
toneMappingExposure  = 0.91 + kickEnv×0.02 + chorusEnv×0.01
```
→ 打击与副歌让整体「发光、过曝」一下。

## 四、一图总结(谁驱动谁)

```
RMS 响度  ─→ levelEnv ─→ 微抖 / 外壳 / 相机角速度 / 整体活跃度
低频能量  ─→ bassEnv  ─→ 引爆力度 / 碎片·烟花数量与速度
8 频段    ─→ bandEnv  ─→ 方块按高度分段:缩放·漂浮·自转·亮度
底鼓 Kick ─→ kickEnv  ─→ 引爆 + 碎片/烟花爆发 + FOV吸气 + Bloom + 方块闪光
节拍 Beat ─→ beatEnv  ─→ 方块颜色轻微闪动
副歌      ─→ chorusEnv─→ 持续撒碎片 + 全局活跃度/泛光/相机收紧
和弦根音  ─→ 颜色索引 ─→ 碎片 / 烟花 / 方块的主色(红/黄/蓝)
整体音量  ─→ motionGate ─→ ★ 总开关:安静=静态海报,响=全场律动
```

> 交互上唯一的「乐器」是音乐本身:插入音频即驱动全场;鼠标点画布可手动触发一次引爆+烟花;暂停会**冻结当前帧**,方便当作静帧海报截图/导出。
